# 喜易云上线包说明

本目录是可上线交付包，已移除 `.git`、`node_modules`、前端 `dist`、后端 `target`、本地 `.env` 等临时或敏感文件。

## 目录用途

- `docker-compose.prod.yml`：生产部署入口。
- `.env.example`：通用环境变量模板。
- `.env.feiniu.example`：飞牛服务器环境变量模板。
- `apps/web`：新版 Web 用户端。
- `apps/admin`：管理员后台。
- `apps/h5`：旧 H5 端。
- `backend`：Spring Boot 后端。
- `docs/`：部署、接口、发布与上线检查文档。

`apps/liquid-next` 为本地 Liquid Glass 设计原型，不属于生产发布范围，生产 Compose 不会构建或启动它。

## 上线步骤

1. 在服务器复制环境变量模板：

```bash
cp .env.feiniu.example .env
```

2. 编辑 `.env`，填写生产密码、后台账号密码哈希、支付回调密钥、卡密加密密钥、端口和正式 HTTPS CORS 域名，并保持 `XIYIYUN_HTTP_BIND=127.0.0.1`。
   `NPM_REGISTRY` 和 `MAVEN_MIRROR_URL` 用于生产镜像构建时拉取依赖，默认使用国内镜像源；服务器访问官方源稳定时可按需调整。

3. 运行上线预检；如果是已有数据库升级，先备份并执行 `db/migrations/002_config_persistence.sql`，再启动新后端：

```bash
npm run prod:preflight -- --env-file .env
npm run launch:verify -- --env-file .env
# 仅已有数据库升级需要执行；全新数据卷跳过这一行
docker compose -p xiyiyun -f docker-compose.prod.yml --env-file .env exec -T mysql \
  sh -lc 'mysql -u"$MYSQL_USER" -p"$MYSQL_PASSWORD" "$MYSQL_DATABASE"' < db/migrations/002_config_persistence.sql
docker compose -p xiyiyun -f docker-compose.prod.yml --env-file .env up -d --build
```

4. 查看服务状态：

```bash
docker compose -p xiyiyun -f docker-compose.prod.yml --env-file .env ps
```

5. 默认端口建议：

- Web 用户端：`WEB_PORT=18080`
- 旧 H5：`H5_PORT=18081`
- 管理后台：`ADMIN_PORT=18088`

生产 Compose 默认把三端 HTTP 端口绑定到 `127.0.0.1`，请通过同机 HTTPS 反向代理对外暴露 Web、H5 和管理后台，不要把裸 HTTP 管理后台直接暴露公网。

## 上线前检查

本包生成前已通过：

```bash
npm run typecheck
docker compose -p xiyiyun run --rm backend mvn test
```

测试结果：前端三端构建通过，后端 57 个测试通过。

## 注意事项

- 不要把真实 `.env`、数据库密码、支付密钥、卡密加密密钥提交到公开仓库。
- 生产环境不要使用 `.env.example` 中的默认密码或示例密钥。
- 真实对外收款前必须配置 HTTPS 反向代理，并将 `/api`、`/ws`、`/uploads` 交给项目内置 Nginx 转发。
- 首次上线前建议完整阅读 `docs/LAUNCH_CHECKLIST.md` 和 `docs/FNOS_DEPLOYMENT.md`。
