# 虚拟商品平台 PRD

## 1. 产品定位

本项目是一个前后端分离的虚拟商品交易平台，面向卡密商品、直充商品和代充商品的在线售卖、履约、支付、会员 API 接入与后台运营。

首版 MVP 聚焦“卡密商品闭环”：商品上架、库存导入、下单支付、自动发卡、订单查询、后台订单与库存管理。直充与代充能力在数据模型和接口边界上预留，但不作为首版核心交付范围。

## 2. 用户角色

| 角色 | 说明 | 核心诉求 |
| --- | --- | --- |
| 游客 | 未登录访问者 | 浏览商品、查看分类与平台过滤结果 |
| 普通用户 | 注册并登录的购买者 | 下单、支付、查看订单、接收卡密或充值结果 |
| 会员/API 用户 | 具备 API 权限的客户 | 通过会员 API 查询商品、创建订单、查询履约状态 |
| 运营人员 | 后台商品与订单操作人员 | 维护分类、商品、库存、订单、导出报表 |
| 财务人员 | 处理支付、退款和对账 | 查看支付流水、发起/审核退款、导出数据 |
| 管理员 | 平台配置与权限负责人 | 管理后台账号、角色权限、审计日志、系统配置 |

## 3. 商品形态

### 3.1 卡密商品

卡密商品是首版 MVP 的核心商品类型。用户支付成功后，系统从库存中强一致锁定并发放卡密。

关键能力：

- 卡密批量导入、去重、校验和状态管理。
- 库存数量实时展示，避免超卖。
- 支付成功后自动发卡。
- 支持已发卡密在订单详情中展示。
- 支持库存预警、库存导出与异常库存排查。

### 3.2 直充商品

直充商品由平台调用上游 API 完成履约。系统需要支持多上游渠道路由、失败重试、任务状态追踪与上游监控。

首版处理策略：

- 建立直充任务表和渠道模型。
- 暂缓复杂路由策略，仅保留接口和数据结构扩展位。
- 后续按渠道优先级、成本、成功率、限流、可用性进行路由。

### 3.3 代充商品

代充商品由人工在后台处理，适用于无法自动化履约的业务。

关键能力：

- 用户下单时收集必要充值信息。
- 后台领取、备注、处理、完成或驳回订单。
- 操作全量写入审计日志。

## 4. 分类、平台与权限

### 4.1 五级分类

平台支持最多五级商品分类：

1. 一级类目：业务大类。
2. 二级类目：平台或服务类型。
3. 三级类目：商品族。
4. 四级类目：规格组。
5. 五级类目：具体销售目录。

分类要求：

- 每个商品挂载到一个最终分类节点。
- 支持后台维护排序、启停、名称、图标与 SEO 字段。
- 用户端分类查询只返回启用节点。
- 后台需要能查看完整分类树。

### 4.2 平台过滤

平台过滤用于按游戏、视频、音乐、工具、运营商或其他业务平台筛选商品。

要求：

- 商品可绑定一个或多个平台标签。
- 用户端支持平台筛选与分类联动。
- 后台支持平台启停和排序。

### 4.3 用户组权限

用户组用于控制可见商品、价格、API 权限和后台能力。

要求：

- 不同用户组可配置不同商品可见性。
- 商品支持按用户组配置价格或折扣。
- API 用户必须显式开通接口权限和限流额度。
- 后台账号采用 RBAC 权限模型，按菜单、按钮、数据范围授权。

## 7. 支付与退款

支付要求：

- 订单创建后生成支付单，支付单与业务订单分离。
- 支付回调必须幂等，按支付单号或第三方交易号去重。
- 支付成功只能触发一次履约。
- 支持主动查询支付状态，用于修复回调丢失。

退款要求：

- 退款单独建模，记录原支付单、退款金额、原因、发起人和状态。
- 退款请求必须幂等，避免重复退款。
- 卡密已发放订单默认不支持自动退款，需要走人工审核。
- 所有退款操作写入审计日志。

## 8. 库存一致性

卡密库存必须强一致，核心目标是避免超卖、重复发卡和库存状态漂移。

要求：

- 卡密记录具有唯一编码或哈希去重。
- 下单支付成功后的发卡过程必须在数据库事务内完成库存锁定与订单绑定。
- 使用行级锁或乐观锁控制并发。
- 卡密状态至少包括：未售、锁定、已售、作废。
- 支持异常补偿任务扫描支付成功但未发卡的订单。

## 9. 非功能需求

- 性能：商品列表和分类数据优先走 Redis 缓存。
- 可用性：支付回调、发卡、导出等任务需要可重试。
- 安全：用户密码加密存储，API 调用签名校验，后台敏感操作二次确认。
- 审计：后台关键操作记录操作者、对象、前后值、IP、时间。
- 可观测：记录订单、支付、履约、上游 API 的结构化日志。
- 导出：订单导出采用异步任务，避免大查询阻塞后台。

## 10. MVP 验收标准

- 后台可创建五级分类、平台标签、卡密商品和用户组权限。
- 后台可批量导入卡密并查看库存状态。
- 用户端可筛选商品、下单并完成支付。
- 支付成功后系统自动发放唯一卡密。
- 同一支付回调重复通知不会重复发卡。
- 并发购买最后一份库存时不会超卖。
- 后台可查询订单详情、导出订单、查看审计日志。
- 支付成功但发卡失败的订单可被补偿任务修复或人工处理。
